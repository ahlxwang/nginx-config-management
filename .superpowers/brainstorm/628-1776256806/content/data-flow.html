核心业务流程 — 配置下发流程

从编辑到生效的完整数据流

1
用户编辑配置
前端页面 → POST /config/edit → Flask 接收请求
↓
2
保存到本地 + 数据库
• 配置文件写入本地目录（按集群分类）
• 配置路径写入 MySQL nginx_config 表
↓
3
用户点击"下发"
前端 → POST /config/deploy → DeployService 启动
全量下发：rsync --delete 同步整个配置目录到目标服务器
↓
4
SSH 批量执行
遍历集群下所有主机（从 hosts 表查询）
• SSH 连接到每台服务器
• rsync 同步配置文件
• 记录结果到 tasklog 表
↓
5
校验配置（可选）
用户点击"校验" → SSH 执行 nginx -t
返回校验结果到前端展示
↓
6
重载 Nginx
用户点击"reload" → SSH 执行 nginx -s reload
配置生效，返回执行结果
↓
7
记录操作日志
所有关键操作（编辑、下发、校验、reload）写入 oplog 表
包含：用户名、集群名、主机名、操作类型、时间戳
关键设计点：
• 全量同步：使用 rsync --delete 确保目标目录与源目录完全一致
• 批量操作：一次下发可同时更新集群内所有服务器
• 操作解耦：下发、校验、reload 分离，用户可灵活控制
• 审计追踪：所有操作记录到 oplog，支持事后审计
