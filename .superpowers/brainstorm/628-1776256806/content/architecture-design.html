系统架构设计 — 三层架构

Nginx 集群配置管理系统

表示层（Presentation Layer）

登录页
配置管理
集群管理
主机管理
用户管理
操作日志
技术：HTML + CSS + JavaScript
↓ HTTP Request/Response

业务逻辑层（Business Logic Layer）

Flask 路由层
• /login - 登录认证
• /config/* - 配置管理
• /cluster/* - 集群管理
• /host/* - 主机管理
• /user/* - 用户管理
• /log/* - 日志查询
核心服务
• ConfigService - 配置文件操作
• DeployService - SSH/rsync 部署
• NginxService - nginx -t / reload
• AuthService - Session 鉴权
• LogService - 操作日志记录
技术：Flask (Python 3.7~3.10)
↓ SQL Query

数据访问层（Data Access Layer）

users
用户表
clusters
集群表
hosts
主机表
nginx_config
配置表
oplog
操作日志
tasklog
任务日志
技术：MySQL 8.0+
↔ SSH/rsync

外部系统（External Systems）

Nginx 服务器 1
SSH 连接
配置同步
nginx -t / reload
Nginx 服务器 2
SSH 连接
配置同步
nginx -t / reload
Nginx 服务器 N
SSH 连接
配置同步
nginx -t / reload
核心数据流：配置下发流程
1. 用户在前端编辑配置文件
2. Flask 后端保存到 MySQL（nginx_config 表）
3. 用户点击"下发"，后端通过 rsync 同步配置到目标服务器
4. 执行 nginx -t 校验配置
5. 校验通过后执行 nginx -s reload
6. 记录操作日志到 oplog 和 tasklog
